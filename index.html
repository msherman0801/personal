National
Disaster Hub

This application was inspired by the rapid growing occurances of natural and manmade disasters across the world. With the purpose of acting as a centralized check-in point for victim's of these disasters, loved ones can search the status of the victim's they know to be affected. A built in API allows for third party's to check in using their own secured systems.

This application uses React, Redux, and Thunk on the front end. A Rails back end acts as both an API internally for the application, as well as externally, for third parties to submit check-in requests. Additionally, multiple third party API's are used.

Wedsites

Wedsites is a wedding website content management system that allows the user to create a beautiful website for their wedding. The website includes an invitation system where invitees can accept their's digitally.

This project was built using Ruby on Rails on the front and back end. Javascript is also used for asynchronous functionality. Additionally, it uses OAuth for users to create an account securely using their facebook credentials.

Dishbuddy

The purpose of Dishbuddy was more-so to display different pieces of functionality, as it was one of the first full-stack web-application's I've created. It allows you to search for top rated restaurants in a specified location and pulls from Zomato's public API to display information for that restaurant. You can save restaurants, add friends, follow their restaurant feeds, and leave reviews.

This application uses Sinatra (Ruby) on both the front and back end. It's also using Active Record for data representation and handling.
